# Печать графика
## Постановка задачи
В рамках лабораторной работы необходимо разработать приложение для печати графиков. Предложен начальный вариант архитектуры ПО, в которую требуется внести изменения с целью снижения связности архитектуры. Необходимо реализовать отображение файлов, из которых выгружаются данные для построения графиков, отрисовку графика в зависимости от выбранного файла, а также возможность выбрать тип графика, сделать график чёрно-белым, сохранить график в PDF формате на устройстве.\
Предлагается работать с данными, которые представляют из себя временных рядов, двух форматов: SQLLite и JSON\
При разработке архитектуры необходимо учесть:\
  Возможность добавления новых видов графиков\
  Изменение визуального стиля графиков.\
Общие требования к GUI:\
Загружаем данные, путем выбора нужного файла. Данные в ПО не отображаем, отображаем только график, построенный относительно считанных данных.
При печати в pdf выбираем место сохранения графика.\
## Предлагаемое решение
### IOC контейнер
Для понижения связности исходной архитектуры предлагается воспользоваться таким элементом как IOC контейнер. Который возьмёт на себя ответственность за создание экземпляра объекта, вынесет его из тела метода за пределы класса и передаст уже созданный экземпляр объекта обратно. Таким образом решится проблема зависимости классов высокого уровня от классов, выполняющих процедуры низкого уровня. \
### Паттерн проектирования "Адаптер"
Также немало важным инструментом в этой работе будет являться структурный паттерн проектирования "Адаптер". Адаптер позволит объектам с несовместимыми интерфейсами работать вместе. Это объект, который трансформирует интерфейс или, в нашем случае, данные одного объекта в такой вид, чтобы он стал понятен другому объекту.\
### MVC
Ещё одна идея, заложенная в решение представленной задачи, заключается в использовании концепции программирования MVC ("модель-вид-контроллер").Основная концепция идеологии MVC – разделить части программы, отвечающие за:\
  Xранение и доступ к данным\
  Их отображение\
  Взаимодействие с пользователем\
на три отдельных модуля, разработка которых относительно независима. В моей реализации используются доступные готовые классы для модели и представления.\
UML диаграмма архитектуры прилагается\
## Инструкция пользователя
После запуска программы открывается окно, которое разделено на две основные части.\
В нижнем левом углу окна представлены две кнопки: "Загрузить Файл" и "Удалить". По нажатию первой кнопки открвается диалоговое окно, где можно выбрать файл для загрузки в приложение. По нажатию второй можете удалить ранее загруженный файл. Если вы попробуете загрузить файл неподдерживаемого формата или пустой, то приложение выведет соответствующую ошибку.\
После добавления файлов их список с некоторыми характеристиками выводится в левой части окна приложения. По нажатию на любой из файлов в правой части окна приложение будет построен график по данным из выбранного файла. Имеется возможность выбрать тип графика, сделать его чёрно-белым или сохранить в формате PDF. За это отвечают кнопки в верхнем правом углу окна. При нажатии на кнопку для печати графика в формате PDF откроется диалоговое окно для выбора места на устройстве для сохранения.\

## Тестирование
В данном разделе описываются различные ситуации действий пользователя и реакция программы на них\
### Case 1
```
Загрузка файла без данных
Приложение выведет окно "Ошибка. Нет данных в: + filePath"
```
### Case 2
```
Загрузка файла c неподдерживаемым форматом данных
Приложение выведет окно "Ошибка. Нет адаптера для: + filePath"
```
### Case 3
```
Загрузка файла неподдерживаемого формата
Приложение выведет окно "Ошибка. Неподдерживаемый формат: + filePath"
```
### Case 4
```
Загрузка одного файла
Приложение выведет название файла и его характеристики в левой части окна
```
### Case 5
```
Загрузка множества файлов
Приложение выведет все названия файлов и их характеристики в левой части окна. При нажатии на любой файл в правой части окна приложения будет построен соответствующий график

Выбор другого типа графика
Приложение перерисует график в правой части окна

Установление галочки напротив "чёрно-белый график"
Приложение перерисует график в правой части окна на чёрно белый

Нажатие на кнопку "Печать PDF"
Приложение откроет диалоговое окно, после выбора места для сохранения график будет экспортирован
```
